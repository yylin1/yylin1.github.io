Portainer TLS認證

Portainer 是一個輕量級的 UI 管理界面，可輕鬆管理 Docker 主機或 Swarm 叢集，遠端管理都會使用到Docker API，預設開啟沒有安全保護的 port 2375 產生連線漏洞，而我們就會需要配置TLS認證憑證，以確保連線安全，此篇會說明如何針對Portainer配置主機環境加入TLS加密。

1. Portainer概述

[GitHub] https://github.com/portainer/portainer
[文檔] https://portainer.readthedocs.io/en/stable/

  • Portainer（基於Go）是一個輕量級的管理界面，可讓您輕鬆管理Docker 主机或Swarm 集群。

  • Portainer的使用意圖是簡單部署。它包含可以在任何Docker引擎上運行的單個容器（Docker for Linux和Docker for Windows）。

  • Portainer允許您管理Docker容器，圖像，音量，網絡等。它與獨立的Docker引擎和Docker Swarm兼容。

限制

Portainer完全支持以下Docker版本：

Docker 1.10到Docker 17.03（包括集群模式）
Docker Swarm> = 1.2.3
對以下Docker版本的部分支持（某些功能可能不可用）：

Docker 1.9

特性

  • 镜像管理
  • 网络管理
  • 数据卷管理
  • 应用模板：支持自定義App模板，例如Nginx，Redis等
  • 状态面板
    • 可查看Docker主機的容器，鏡像，音量，網絡（包括查看容器內部IP）
    • 容器狀態監控：CPU，內存，網絡利用率，每7秒刷新一次
    • 容器運行時狀態：啟動命令，環境變量等
  • 控制台：
    • Web UI提供容器的控制台界面
  • 日誌：可實時查看容器運行日誌
  • 权限分配：針對端點（Docker主機）分配權限

使用感受（納入生態圈）

小而美，目前只支持Docker和Swarm集群的管理
基於tcp的連接方式，可在一台主機上，對所有Docker宿主機進行管理
對於mesos + marathon的PaaS平台來說，做為擴展項目相當不錯
目前不支持LDAP，一大遺憾……

使用docker-machine部署Docker engine主機時，會為每一台主機生成服務器證書
因此，在使用portainer管理Docker主機，在添加Endpoints時，需要開啟TLS認證，上傳相關docker-mahcine部署主機（192.168.20.4）的證書和key

  • TLS CA certificate：ca.pem
  • TLS certificate：cert.pem
  • TLS key：key.pem

新增 Endpoints 主機

為了安全性關係，在使用 portainer 管理Docker主機，在新增 Endpoints 時，需要開啟 TLS 認證，上傳 docker-mahcine 部署主機的憑證和key：

  • TLS CA certificate：ca.pem
  • TLS certificate：cert.pem
  • TLS key：key.pem
