透過 Ansible 快速部署安 Openshift 4.0 叢集

本篇將介紹透過 OpenShift Ansible 來快速部署實體主機Openshift 4.0叢集版本，這邊主要透過安裝最新版本(v4.0)依賴套件來部署裸機(Bare machines)過程。

共同編輯: chlin62

Openshift 由 Red Hat 公司推出的一個基於主流的容器技術 Docker 和 kubernetes 構建的的開源容器應用平台。Openshift 底層以 Docker 作為容器引擎驅動，以 Kubernetes 作為容器編排引擎組件，並提供了開發語言、中間件、DevOps自動化流程工具和 Web console 用戶介面等元素，提供了一套完整的基於容器的PaaS應用雲平台。

本篇將介紹透過 OpenShift Ansible 來快速部署實體主機 Openshift 4.0叢集版本，這邊主要透過安裝最新版本(v4.0)依賴套件來部署裸機(Bare machines)過程。

本次 Openshift 安裝版本：

  • Openshift Community v4.0
    • [OKD: The Origin Community Distribution of Kubernetes]

OpenShift 部署可以分為以下幾個階段：

1、環境事前準備：準備OpenShift叢集需要的主機，並進行事前環境相關連結配置。

2、安裝依賴套件：提前安裝使用Ansible安裝OpenShift叢集所依賴的最新軟版本.rpm套件。

3、Ansible執行安裝：安裝使用Ansible Playbook進行自動化安裝。

4、OpenShift系統配置：在使用Ansible執行安裝完成之後的系統配置。

5、測試與額外套件 : 測試部署後的環境與額外add-on套件於叢集環境。

實驗節點資訊

本次安裝作業系統採用 CentOS 7.5，測試環境為實體主機：

事前準備

安裝前需要確認以下幾個項目：

  • 所有節點的網路之間可以互相溝通。
  • 部署節點對其他節點不需要 SSH 密碼即可登入。
  • 所有節點都擁有 Sudoer 權限，並且不需要輸入密碼。
  • 所有節點需要安裝 Python。
  • 所有節點需要設定/etc/host解析到所有主機。
  • 部署節點需要安裝 Ansible。

建議部署這邊可以透過一台在同網域的basion主機直接操作多台主機環境

設置主機名

為方便識別主機對應名稱，這邊修改每台實體機hostname

Network Setting 設置 Domain

安裝 CentOS Linux 7 作業系統時，網路透過CLI設定配置，而這邊需要先配置好Openshift訪問網頁需要配置Domain。

ifcfg-em1 是網卡名稱，如果是 eth0 修改對應網卡 ifcfg-eth0

每台節點編輯配置網路設定，並在最後面加上自己的 DOMAIN="xxx.xx.xx"。

設定完後重啟 NetworkManager

Ref: [How can I add additional search domains to the resolv.conf created by dhclient in CentOS]

設置 /etc/hosts 配置

為部署方便，讓節點網路之間可以互相溝通，在/etc/hosts中添加了節點資訊縮寫，來解析各主機。

  • 所有節點需要設定/etc/hosts解析到所有叢集主機。

這邊直接將Master節點設定配置/etc/hosts傳送給其他節點

設置免 SSH 密碼

為方便部署節點對其他節點免密碼即可登入，這邊直接透過Master 節點複製public key給其他節點。

  • 參考: 實現免密碼 ssh 登入遠端主機

檢查SELINUX是否開啟

修改方式如下：

安裝依賴套件 (Install dependency packages)

  • 所有的主機下執行以下命令安裝OpenShift依賴套件包

安裝 Docker (Install docker)

  • 所有節點都需要執行安裝docker

安裝本地儲存庫 (Install local repostory)

安装依賴套件（Install openshit 4.0 alpha depency package）

這邊移動套件來源路徑

  • 這邊需要一個一個下載依賴套件到資料夾裡，如果安裝套件.rpm需要在離線環境時時，可以先透過可以連線Internet機器下載完後，再scp進去至離線環境

  • 連結下載：[Download openshift-origin-v4.0 depency package]

master節點安裝方式使用 wget xxx.rpm，或是網頁點選下載rpm檔後，用rpm指令安裝

使用命令createrepo將現有的剛剛下載好的origin-4.0.rpm創建為自定義的yum倉庫源

如果由 Master節點 當 basion 機時，必須要修改預設的 firewall，要不然node會抓不到東西

編輯套件庫 (Edit repos）

  • 這邊由Master節點複製 openshift 套件包傳遞給所有節點

確認預設 Openshift Repository 狀態

  • CentOS-Base.repo

  • CentOS-OpenShift-Origin.repo

  • epel-testing.repo

  • epel.repo

  • Master節點啟用EPEL倉庫以安裝Ansible

  • 從 github 上clone openshift-ansible 4.0版本
